El modulo **CCP** (*Capture / Compare / PWM*) tiene tres modos; el mas usado en el curso es **PWM** — control de **velocidad de motores**, **dimmer de LEDs** y **servo**. La base de tiempo es **[Timer2 + PR2](/timers/timer1-timer2/#timer-2--periodo-fijo-y-pwm)**, no Timer0.

Esta pagina cierra **Tema 9 CCP** (PWM): formulas del PDF, registros y ejemplo **5 kHz al 50 %** con cristal **20 MHz**. Aplicacion en [Practica 4](/tutoriales/practica-4/) y [Practica 6 — Motor paso a paso](/tutoriales/practica-6/).

<AIExplain title="Pines CCP">
**CCP1** → **RC2**. **CCP2** → **RC1**. En modo PWM la salida la genera el hardware; tu cargas **PR2** (periodo) y **CCPR1L** (duty).
</AIExplain>

## Que es PWM

**PWM** (*Pulse Width Modulation*) repite una onda digital: el pin pasa de **alto** a **bajo** con periodo fijo **T_PWM**. Cambias el **duty cycle** — porcentaje del periodo en alto — para simular potencia analogica en motor o LED.

### Paso 1 — Periodo y duty

| Parametro | Simbolo | Que controla |
| --- | --- | --- |
| **Periodo** | T_PWM | Frecuencia F_PWM = 1/T_PWM |
| **Duty cycle** | X % | Tiempo en alto / T_PWM |
| **Resolucion** | bits efectivos | Cuántos pasos de duty puedes programar |

### Paso 2 — Tres modos del CCP

| Modo | Timer base | Uso |
| --- | --- | --- |
| **Captura** | Timer1 | Medir ancho de pulso / frecuencia |
| **Comparacion** | Timer1 | Generar flanco a instante preciso |
| **PWM** | **Timer2 + PR2** | Motor, LED, servo — **foco del curso** |

Captura/comparacion se ven en [Timer 1](/timers/timer1-timer2/#timer1-en-ccp--captura-y-comparacion); PWM es independiente con **Timer2**.

### Paso 3 — Registros clave en PWM

| Registro | Funcion |
| --- | --- |
| **`PR2`** | Periodo — TMR2 cuenta 0 → PR2 → reset |
| **`T2CON`** | Prescaler Timer2, **TMR2ON** |
| **`CCPR1L`** + **`CCP1CON<5:4>`** | Duty cycle (10 bits en PIC18) |
| **`CCP1CON`** | Modo CCP — PWM = `CCP1M3:0 = 1100` → **`0x0C`** |

### Paso 4 — Salida en hardware

Con `CCP1CON = 0x0C` y Timer2 activo, **RC2** conmuta solo — no hace falta `BSF LATC, 2` en bucle. Configura **`BCF TRISC, 2`** (salida) una vez al inicio.

## Formulas del Tema 9

**Periodo PWM:**

```
  T_PWM = 4 × (PR2 + 1) × Tosc × Prescaler_Timer2
```

**Duty cycle (valor a cargar):**

```
  CCPR1L : CCP1CON<5:4> = (T_PWM × X%) / (Tosc × Prescaler)
```

**Resolucion (bits efectivos):**

```
  log2( Fosc / (F_PWM × Prescaler) )
```

Con **Fosc = 20 MHz**, **prescaler Timer2 = 1:4**, **F_PWM = 5 kHz**:

```
  PR2 = 1 / (5 kHz × 4 × Tosc × 4) − 1 = 249
  Duty 50 % → CCPR1L = 124
```

## Configuracion PWM — 5 pasos

1. **`BCF TRISC, 2`** — RC2/CCP1 como salida
2. Cargar **`PR2`** segun frecuencia deseada
3. Configurar **`T2CON`** (prescaler, **TMR2ON = 1**)
4. Cargar **`CCPR1L`** y bits **DC1B1:DC1B0** en `CCP1CON` si aplica
5. **`CCP1CON = 0x0C`** — modo PWM

## Ejemplo completo — 5 kHz, 50 % duty @ 20 MHz

```asm
config_pwm_5khz_50:
        BCF     TRISC, 2            ; RC2 salida (CCP1)
        MOVLW   D'249'
        MOVWF   PR2                 ; periodo ~5 kHz, PS=1:4
        MOVLW   D'124'
        MOVWF   CCPR1L              ; duty ~50 %
        MOVLW   B'00001101'         ; TMR2ON, prescaler 1:4
        MOVWF   T2CON
        MOVLW   0x0C                ; CCP1 modo PWM
        MOVWF   CCP1CON
        RETURN
```

Para cambiar velocidad de motor: modifica **`CCPR1L`** (duty) sin recalcular PR2 si mantienes la misma frecuencia PWM.

## CCP1 vs CCP2

| Modulo | Pin | Practica |
| --- | --- | --- |
| **CCP1** | RC2 | Practica 4 PWM, Practica 6 motor |
| **CCP2** | RC1 | Segunda salida PWM si el diseno lo requiere |

Ambos en modo PWM comparten **Timer2** y **PR2** — misma frecuencia, distinto duty en `CCPR1L` / `CCPR2L`.

## Errores comunes

| Error | Consecuencia |
| --- | --- |
| PWM sin **Timer2ON** | RC2 quieto |
| Olvidar **TRISC** | Pin no conduce |
| **`CCP1CON` en captura** | No hay onda PWM |
| PR2 mal calculado | Frecuencia distinta a la esperada |
| Mezclar Timer1 y PWM | Captura usa T1; **PWM usa T2** |

## Preguntas estilo parcial

**¿Que timer usa el PWM del CCP?**

**Timer2** con registro de periodo **PR2**.

**¿Que valor tipico pone CCP1 en modo PWM?**

**`CCP1CON = 0x0C`** (bits modo 1100).

**¿En que pin sale CCP1?**

**RC2**.

**Para 50 % duty con PR2 = 249, CCPR1L ≈ ?**

**124** (mitad del periodo en cuentas de duty).
